### `WS /ws/chat/{case_id}`

**Defined in:** `backend/api/chat.py` (handler: `websocket_chat_endpoint`)
**Auth:** none
**Path params:** `case_id` (string) — case ID, e.g. `ACTE-2024-001`
**Query params:** `language` (string, default `de`) — `de` or `en`

This is a WebSocket endpoint, not a REST endpoint. After connection, the server first sends a system welcome message (`{"type": "system", "content": "...", "timestamp": null}`), then the client may send JSON messages.

**Incoming message types (client → server):**
- `chat` — chat message (content, optional documentContent, formSchema, folderId, currentFormValues, stream, language)
- `anonymize` — request anonymization (filePath, documentId, folderId, language)
- `translate` — request translation (filePath, targetLanguage, sourceLanguage, documentId)

**Outgoing message types (server → client):**
- `system` — welcome / informational
- `chat_response` — non-streaming AI reply
- `chat_chunk` — streaming AI reply chunk (with `is_complete` flag)
- `form_update` — form field updates with confidence
- `form_suggestion` — suggestion-mode field proposals
- `anonymization_complete` — anonymization result. When `ENABLE_ANONYMIZATION=false` (Wave 3 / S001-F-004), the server short-circuits and emits `{success: false, error: "feature_disabled", message: <localized DE/EN string>, timestamp: null}` immediately — no upstream anonymization service is contacted. Localized message is `"Die Anonymisierungsfunktion ist in dieser Demo-Umgebung noch nicht verfügbar."` (de) or `"The anonymization feature is not yet available in this demo environment."` (en).
- `translation_complete` — translation result
- `error` — error description

**Example incoming message:**
<!-- example source: synthesis from docstring -->
```json
{
  "type": "chat",
  "content": "User message",
  "caseId": "ACTE-2024-001",
  "folderId": "personal-data",
  "documentContent": "...",
  "formSchema": []
}
```

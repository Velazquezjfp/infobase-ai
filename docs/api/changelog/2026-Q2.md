# API Changelog — 2026 Q2

## 2026-04-27

Genesis entry — first run of the API documenter. All endpoints currently exposed by the backend are listed below as Added.

### Added

- `GET /` — Root endpoint returning API name and version. Defining file: `backend/main.py`.
- `GET /api/admin/health` — Admin service health check (field generation + form modification readiness). Defining file: `backend/api/admin.py`.
- `POST /api/admin/generate-field` — Generate a SHACL-compliant form field from a natural-language prompt (rule-based + Gemini fallback). Defining file: `backend/api/admin.py`.
- `POST /api/admin/modify-form` — Apply natural-language form modifications, returning updated fields, a SHACL NodeShape, and applied modifications (S5-001). Defining file: `backend/api/admin.py`.
- `POST /api/chat/clear/{case_id}` — Clear stored conversation history for a case (S5-010, requires `ENABLE_CHAT_HISTORY`). Defining file: `backend/api/chat.py`.
- `GET /api/chat/health` — Chat service health check (Gemini initialization status). Defining file: `backend/api/chat.py`.
- `GET /api/context/case/{case_id}` — Retrieve full case context including all folder contexts for AI transparency. Defining file: `backend/api/context.py`.
- `GET /api/context/tree/{case_id}` — Hierarchical document tree view for a case (S5-011 cascading context). Defining file: `backend/api/context.py`.
- `GET /api/custom-context/{case_id}` — List all custom validation rules and required documents for a case (S5-017). Defining file: `backend/api/custom_context.py`.
- `POST /api/custom-context/{case_id}/document` — Add a custom required-document rule to a case. Defining file: `backend/api/custom_context.py`.
- `POST /api/custom-context/{case_id}/rule` — Add a custom validation rule to a case. Defining file: `backend/api/custom_context.py`.
- `DELETE /api/custom-context/{case_id}/{rule_id}` — Remove a custom rule (validation or required-document) from a case. Defining file: `backend/api/custom_context.py`.
- `GET /api/documents/all` — Retrieve all documents across all cases (admin/debug view). Defining file: `backend/api/documents.py`.
- `POST /api/documents/extract-pdf-text` — Extract full text and page count from a PDF document (S5-003). Defining file: `backend/api/documents.py`.
- `GET /api/documents/health` — Document registry service health check. Defining file: `backend/api/documents.py`.
- `POST /api/documents/parse-email` — Parse a `.eml` file and return headers, body (text + HTML), and attachment metadata (S5-008). Defining file: `backend/api/documents.py`.
- `GET /api/documents/tree/{case_id}` — Document tree (folders + documents + renders) for a case, with localized folder names. Defining file: `backend/api/documents.py`.
- `GET /api/files/exists/{case_id}/{folder_id}/{filename}` — Check whether a file already exists; suggests a unique filename if it does. Defining file: `backend/api/files.py`.
- `GET /api/files/health` — File service health check (storage path availability). Defining file: `backend/api/files.py`.
- `DELETE /api/files/renders/{case_id}/{document_id}/{render_id}` — Delete a specific document render (anonymized, translated, etc.) — S5-006. Defining file: `backend/api/files.py`.
- `POST /api/files/upload` — Upload a file (multipart) into `public/documents/{case_id}/{folder_id}/`, with size limit, sanitization, and registry registration. Defining file: `backend/api/files.py`.
- `DELETE /api/files/{case_id}/{folder_id}/{filename}` — Delete a file from a case folder with path-traversal protection. Defining file: `backend/api/files.py`.
- `GET /api/folders/{case_id}` — Get the folder configuration for a case (creates a default config if missing). Defining file: `backend/api/folders.py`.
- `POST /api/folders/{case_id}` — Create a new folder for a case (config entry + physical directory). Defining file: `backend/api/folders.py`.
- `PUT /api/folders/{case_id}` — Bulk replace all folders for a case (admin-panel save). Defining file: `backend/api/folders.py`.
- `POST /api/folders/{case_id}/reorder` — Reorder folders by providing the desired list of folder IDs. Defining file: `backend/api/folders.py`.
- `DELETE /api/folders/{case_id}/{folder_id}` — Delete a folder; non-empty folders require `?force=true` (contents moved to `uploads`). Defining file: `backend/api/folders.py`.
- `PUT /api/folders/{case_id}/{folder_id}` — Update a folder's localized name, mandatory flag, or display order. Defining file: `backend/api/folders.py`.
- `POST /api/idirs/rag` — Proxy a RAG query to IDIRS over a list of doc IDs, then enrich with Gemini analysis and a confidence score. Defining file: `backend/api/idirs.py`.
- `POST /api/idirs/search` — Proxy hybrid (BM25 + kNN) search to the IDIRS OpenSearch service. Defining file: `backend/api/idirs.py`.
- `GET /api/search/health` — Semantic-search service health check. Defining file: `backend/api/search.py`.
- `POST /api/search/semantic` — Gemini-powered semantic search with cross-language support (S5-003). Defining file: `backend/api/search.py`.
- `POST /api/validation/case/{case_id}` — AI-powered case validation: returns score, summary, warnings (by severity), and recommendations. Defining file: `backend/api/validation.py`.
- `GET /api/validation/health` — Case-validation service health check. Defining file: `backend/api/validation.py`.
- `GET /health` — Top-level liveness probe. Defining file: `backend/main.py`.
- `WS /ws/chat/{case_id}` — Case-scoped WebSocket for AI chat, form extraction, anonymization, and translation. Accepts `language` query param. Defining file: `backend/api/chat.py`.

## 2026-04-28

### Modified
- `GET /api/chat/health` — response payload gained an `"anonymization"` key (`"enabled"` | `"disabled"`) reflecting the new `ENABLE_ANONYMIZATION` feature flag (Wave 3 / S001-F-004). Defining file: `backend/api/chat.py`.
- `WS /ws/chat/{case_id}` — `anonymization_complete` outgoing message now has a distinct disabled-feature variant: when `ENABLE_ANONYMIZATION=false`, the server returns `{"type": "anonymization_complete", "success": false, "error": "feature_disabled", "message": <localized DE/EN string>, "timestamp": null}` immediately, before any anonymization service is contacted. Defining file: `backend/api/chat.py`.

## 2026-04-29

### Modified
- `POST /api/idirs/search` — added early-return `503` with body `{error: "feature_disabled", detail: <localized DE string>}` when `ENABLE_DOCUMENT_SEARCH=false` (S001-F-005). The guard fires before any outbound TCP connection to IDIRS. Previously `503` was reserved for upstream connection failure.
- `POST /api/idirs/rag` — same `503 feature_disabled` short-circuit when `ENABLE_DOCUMENT_SEARCH=false` (S001-F-005).
- `GET /api/search/health` — response now includes `document_search` field (`"enabled"` or `"disabled"`) reflecting the `ENABLE_DOCUMENT_SEARCH` flag (S001-F-005).

### Added
- Schema `ErrorResponse` in `backend/api/idirs.py` — fields: `error: str`, `detail: str`. Used as the body shape for the new `503 feature_disabled` responses on the IDIRS endpoints.

## Feature Flags

- **Required:** no (default: `false`)
- **Type:** boolean (parsed via `get_bool_env`; truthy values: `true`, `1`, `yes`, `on`)
- **Used by:** `backend/config.py`
- **Example value:** `false`
- **Purpose:** enables the PII anonymization service path (S001-F-004). When `false`, the application surfaces a user-facing notice instead of calling the external anonymization service (which is unreachable in the closed environment).

### `ENABLE_DOCUMENT_SEARCH`
- **Required:** no (default: `false`)
- **Type:** boolean (parsed via `get_bool_env`; truthy values: `true`, `1`, `yes`, `on`)
- **Used by:** `backend/config.py` (consumed by `backend/api/idirs.py` and `backend/api/search.py`)

Also surfaces in `GET /api/search/health` as `document_search: "enabled"|"disabled"`.

## LiteLLM

### `LITELLM_MODEL`
- **Required:** no (default: `gemma3:12b`)
- **Type:** string
- **Used by:** `backend/config.py`
- **Example value:** `gemma3:12b`
- **Purpose:** model identifier the LiteLLM proxy should route to (e.g. an Ollama model id on the closed-environment host). Only consulted when `LLM_BACKEND=internal`.

### `LITELLM_PROXY_URL`
- **Required:** no at the config layer (default: `''` empty string), but **enforced as required at provider construction** when `LLM_BACKEND=internal`. `backend/services/llm_provider.py::LiteLLMProvider.__init__` raises `ValueError` when the value is empty and the internal backend is selected (fail-fast precondition).
- **Type:** URL
- **Used by:** `backend/config.py` (read), `backend/services/llm_provider.py::LiteLLMProvider` (validated)
- **Example value:** `http://localhost:4000`
- **Purpose:** OpenAI-compatible base URL of the LiteLLM proxy.

### 🔒 `LITELLM_TOKEN`
- **Required:** no (default: `''` empty string) — sometimes required by the proxy; an empty string is acceptable when the proxy is unauthenticated.
- **Type:** string
- **Used by:** `backend/config.py`
- **Example value:** `sensitive — see secrets manager or .env.example`
- **Purpose:** bearer token / master key presented to the LiteLLM proxy. Must equal the LiteLLM container's `LITELLM_MASTER_KEY`.

### 🔒 `LITELLM_MASTER_KEY`
- **Required:** **yes** (no default) — read inside the `litellm/` Docker subproject; the proxy container fails to authenticate requests without it.
- **Type:** string
- **Used by:** the LiteLLM proxy container (`litellm/config.yaml::general_settings.master_key = os.environ/LITELLM_MASTER_KEY`); cross-referenced by `docs/tests/S001-NFR-004/TC-S001-NFR-004-04.py`, `TC-S001-NFR-004-05.py`, `TC-S001-NFR-004-08.py` (read via `os.environ.get("LITELLM_MASTER_KEY", "sk-bamf-local-dev-key")`).
- **Example value:** `sensitive — see secrets manager or litellm/.env.example`
- **Purpose:** master token enforced by the LiteLLM proxy. Lives in the gitignored `litellm/.env` (not the app `.env`). Must be set to the same string as the backend's `LITELLM_TOKEN` so the backend can authenticate against the proxy. Introduced by S001-NFR-004.

### `LLM_BACKEND`
- **Required:** no (default: `internal`)
- **Type:** enum (`internal` | `external`)
- **Used by:** `backend/config.py` (consumed by `backend/services/llm_provider.py::get_provider()`)
- **Example value:** `internal`
- **Purpose:** selects the LLM provider implementation. `internal` routes through the LiteLLM proxy (closed-environment path; the safe default); `external` calls Google Gemini directly. Any other value raises `ValueError` on first `get_provider()` call.

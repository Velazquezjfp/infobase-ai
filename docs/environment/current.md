## Feature Flags

### `ENABLE_ANONYMIZATION`
- **Required:** no (default: `false`)

### `ENABLE_DOCUMENT_SEARCH`
- **Required:** no (default: `false`)
- **Type:** boolean (parsed via `get_bool_env`; truthy values: `true`, `1`, `yes`, `on`)
- **Used by:** `backend/config.py` (consumed by `backend/api/idirs.py` and `backend/api/search.py`)
- **Example value:** `false`
- **Purpose:** gates IDIRS document search and RAG (S001-F-004 / S001-F-005). When `false`, both `POST /api/idirs/search` and `POST /api/idirs/rag` early-return `503` with `{"error": "feature_disabled", ...}` before opening any outbound TCP connection to IDIRS; the frontend catches this shape and renders a localized "not yet available" notice. Also surfaces in `GET /api/search/health` as `document_search: "enabled"|"disabled"`.

### `ENABLE_UPLOAD`
- **Required:** no (default: `false`)
- **Type:** boolean (parsed via `get_bool_env`; truthy values: `true`, `1`, `yes`, `on`)
- **Used by:** `backend/config.py`
- **Example value:** `false`
- **Purpose:** enables the file-upload subsystem (S001-F-006). When `false`, upload UI is hidden and a user-facing notice is shown instead. Stubbed out for sprint 1.

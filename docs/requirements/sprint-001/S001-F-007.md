# Ephemeral one-shot session with root_docs reset on logout, close, or 10-min idle

## Description

The sprint 1 demo is a single-session experience. The user logs in, uses the chat interface and slash commands, and on any of three events the application must reset every per-case mutable artifact back to the seed state captured under `root_docs/`:

1. The user clicks Logout (`src/pages/Login.tsx`-driven flow; logout currently navigates back to `/login` via the `setUser(null)` flow in `src/contexts/AppContext.tsx::setUser`, line 102).
2. The user closes the browser tab/window — `beforeunload` event fires.
3. The user is idle for more than `SESSION_IDLE_TIMEOUT_MINUTES` (default `10`). Idle is defined as: no `mousemove`, `keydown`, `scroll`, `click`, or WebSocket activity from the client side.

"Reset" means: every file under `${DOCUMENTS_PATH}/${case_id}/` whose path or filename is not present in the `root_docs/` mapping is deleted; every render (`__anonymized` / `__translated_*` suffix) is deleted; the `document_manifest.json` is regenerated by `reconcile()` after the deletion; and any `case.json`, `folder_config.json`, `custom_rules.json`, or per-folder `*.json` under `backend/data/contexts/cases/${case_id}/` is reset back to the corresponding template under `backend/data/contexts/templates/${caseType}/`.

The `root_docs/` seed mapping for sprint 1 is exactly:

| Folder              | Files                                                                                                       |
|---------------------|-------------------------------------------------------------------------------------------------------------|
| `personal-data/`    | `Aufenthalstitel.png`, `Geburtsurkunde.jpg`, `Personalausweis.png`                                          |
| `certificates/`     | `Sprachzeugnis-Zertifikat.pdf`                                                                              |
| `applications/`     | `Anmeldeformular.pdf`                                                                                       |
| `evidence/`         | `Notenspiegel.pdf`                                                                                          |

Note: `Notenspiegel.pdf` was originally categorized as "Additional information" in the input, but the input's "Additional information" folder does not exist in the current default folder set (`personal-data`, `evidence`, `emails`, `certificates`, `applications` — see `docs/database/current.md` line 171). Per user direction, the file is mapped to the existing `evidence/` folder rather than introducing a new default folder. No `folder_config.json` change is required.

A new backend endpoint `POST /api/session/reset` triggers the reset for a given `case_id`. It is invoked by:

- The frontend on `setUser(null)` (logout).
- The frontend on `window.beforeunload` (using `navigator.sendBeacon` so the call is fire-and-forget on tab close).
- The backend's own idle watchdog: a per-case in-memory timer (in `AppContext` on the frontend, plus a server-side guard) tracks last-activity timestamp on the WebSocket connection. When the WebSocket closes and no new activity occurs for `SESSION_IDLE_TIMEOUT_MINUTES`, the backend invokes the same reset logic for that case.

A new service `backend/services/session_manager.py` owns the reset logic — copying `root_docs/` files into `${DOCUMENTS_PATH}/${case_id}/<folder>/`, restoring the templates under `backend/data/contexts/cases/${case_id}/`, and calling `document_registry.reconcile()` to rebuild the manifest.

`backend/main.py::lifespan` (line 46) is extended to perform the same reset for the default case `ACTE-2024-001` on startup, replacing the current `INIT_TEST_DOCS` behaviour. `INIT_TEST_DOCS` is kept for backward compatibility but its scope is documented as "no-op when `ENABLE_SESSION_RESET=true`"; sprint 1 sets `ENABLE_SESSION_RESET=true` implicitly via the new flow.

This requirement deliberately changes the persistence story documented in `docs/database/current.md`: case context, folder config, and custom rules are no longer persistent across sessions. The change is in line with the input's "I need a one shot demo" framing.

## Acceptance criteria

- A new directory `root_docs/` exists at the repository root (already present in the repo with the listed files; no new files needed there).
- A new constant `ROOT_DOCS_MAPPING` in `backend/services/session_manager.py` enumerates the six files above and their target folders.
- A new function `backend/services/session_manager.py::reset_case_to_seed(case_id)` performs the documented reset: clear `${DOCUMENTS_PATH}/${case_id}/`, copy from `root_docs/` to the matching folders, reset `backend/data/contexts/cases/${case_id}/` from the template, and call `document_registry.reconcile()`.
- A new endpoint `POST /api/session/reset` exists in a new file `backend/api/session.py` and accepts `{ "case_id": "ACTE-2024-001" }`; returns `200 {"success": true}` on success.
- `backend/main.py::lifespan` calls `reset_case_to_seed("ACTE-2024-001")` on startup.
- `src/contexts/AppContext.tsx::setUser` calls `POST /api/session/reset` when transitioning user from non-null to null (logout).
- A new `useIdleTimeout(minutes)` React hook in `src/hooks/useIdleTimeout.ts` listens for `mousemove`, `keydown`, `scroll`, `click`, and resets a timer; when the timer fires it calls `setUser(null)` and navigates to `/login`. The hook reads `SESSION_IDLE_TIMEOUT_MINUTES` via the standard `import.meta.env.VITE_SESSION_IDLE_TIMEOUT_MINUTES` (Vite-prefixed copy of the env var), default `10`.
- A `beforeunload` listener registered in `AppProvider` calls `navigator.sendBeacon` against `/api/session/reset` with the active `case_id` payload.
- New i18n key `session.idleTimeout` (`"Sitzung wegen Inaktivität abgemeldet"`) exists in both locale files.
- After running through a full demo session (login → upload-attempts → chat → logout), restarting the app and logging back in shows exactly the seed file set under `${DOCUMENTS_PATH}/ACTE-2024-001/` — no leftover anonymized renders, no leftover translations.

## Test cases

- **TC-S001-F-007-01** (happy path, logout): start the app, login, anonymize a document (with `ENABLE_ANONYMIZATION=true` for the test), logout; `${DOCUMENTS_PATH}/ACTE-2024-001/personal-data/` contains exactly the three seed files and no `__anonymized` renders.
- **TC-S001-F-007-02** (browser close): start the app, modify a custom rule via `POST /api/custom-context/{case_id}/rule`, close the browser tab; after the `sendBeacon` lands, `backend/data/contexts/cases/ACTE-2024-001/custom_rules.json` is reset to the empty template (or absent, if the template provides no defaults).
- **TC-S001-F-007-03** (idle timeout): mock `Date.now`/timers in a Vitest run, render `AppProvider` with `SESSION_IDLE_TIMEOUT_MINUTES=1`, advance time by 61 seconds without firing any input events; the `useIdleTimeout` hook calls `setUser(null)` and navigates to `/login`.
- **TC-S001-F-007-04** (idle reset on activity): same setup, fire a `mousemove` event at 30 seconds; the timer resets and the navigation does not happen at 61 seconds.
- **TC-S001-F-007-05** (evidence folder seeded): after `reset_case_to_seed("ACTE-2024-001")`, `${DOCUMENTS_PATH}/ACTE-2024-001/evidence/Notenspiegel.pdf` exists.
- **TC-S001-F-007-06** (manifest reconciliation): `document_manifest.json` after the reset contains exactly six entries, one per seed file, with fresh `documentId` and `fileHash` values.
- **TC-S001-F-007-07** (lifespan startup): a fresh container with no existing `${DOCUMENTS_PATH}/ACTE-2024-001/` contents starts up, runs `lifespan`, and ends in the seeded state.

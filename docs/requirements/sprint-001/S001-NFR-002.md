# Containerize the Vite frontend with Artifactory-friendly Dockerfile

## Description

The frontend (Vite + React + TypeScript) currently has no Dockerfile. For BDOP it must run from an image based on `${DOCKER_REGISTRY}/node:18-alpine`, with `npm install` going through `${NPM_REGISTRY}` (Artifactory npm proxy in production, public registry in dev). Resource target: 1 vCPU / 1 GiB RAM.

Add `frontend/Dockerfile` (path is at the repo root since the frontend code lives in `src/` and the build artifacts go to `dist/`; we use `frontend/Dockerfile` so the per-component split is explicit, even though both stages COPY from the project root). Use a multi-stage build:

**Stage 1 — `builder`:**

1. `FROM ${DOCKER_REGISTRY}/node:18-alpine AS builder`.
2. `WORKDIR /app`.
3. Accepts `ARG NPM_REGISTRY=https://registry.npmjs.org` and runs `npm config set registry "${NPM_REGISTRY}"`.
4. Copies `package.json` and `package-lock.json` first; runs `npm ci`.
5. Copies `src/`, `public/` (excluding `public/documents/`), `index.html`, `tsconfig*.json`, `vite.config.ts`, `tailwind.config.ts`, `postcss.config.js`, `components.json`.
6. Accepts `ARG VITE_API_BASE_URL=http://localhost:8000` and `ARG VITE_SESSION_IDLE_TIMEOUT_MINUTES=10`; both are exported as env so Vite bakes them into `dist/`.
7. Runs `npm run build` (produces `dist/`).

**Stage 2 — runtime:**

1. `FROM ${DOCKER_REGISTRY}/node:18-alpine`.
2. Installs `serve` (or uses `node` with a tiny static file handler — pin to a stable static-file server; `serve@14.x` is reasonable).
3. Copies `--from=builder /app/dist /app/dist`.
4. Sets `WORKDIR /app/dist`.
5. Runs as non-root user `app` (UID 10002 to differ from backend's 10001).
6. `EXPOSE 3000`.
7. Runs `serve -s . -l 3000` (or equivalent), responding to `GET /` with the Vite-built `index.html`.
8. Adds a `HEALTHCHECK` that fetches `/` and expects a `200`.

A separate `.dockerignore.frontend` is **not** added; the project-root `.dockerignore` (S001-NFR-001) already excludes the patterns needed (`node_modules/`, `dist/`, etc.) and is shared by both Dockerfiles.

The frontend container does not need to reach the backend at build time (build is static). At runtime it only serves static assets; the browser talks directly to the backend at `${VITE_API_BASE_URL}` (which the operator points at the BDOP backend). For local Compose runs, `VITE_API_BASE_URL=http://localhost:8000` (host-side) is sufficient — Compose networking is not used by the browser, only by inter-container communication, which the frontend does not initiate.

## Acceptance criteria

- A file `frontend/Dockerfile` exists with the two-stage build documented above.
- `docker build -t bamf-frontend:dev --build-arg DOCKER_REGISTRY=docker.io --build-arg NPM_REGISTRY=https://registry.npmjs.org --build-arg VITE_API_BASE_URL=http://localhost:8000 -f frontend/Dockerfile .` succeeds.
- Final image size is under 200 MiB.
- The image runtime entrypoint serves static files on port 3000.
- `curl -fsS http://localhost:3000/` against the running container returns `200` and the body contains `<title>` from `index.html`.
- The build stage installs the exact dependencies in `package-lock.json` (`npm ci` not `npm install`).
- Running `docker run -d -p 3000:3000 bamf-frontend:dev` and curling `/assets/<some-bundle>.js` returns `200`.
- `docker inspect bamf-frontend:dev --format '{{.Config.User}}'` returns `app`.
- The image does **not** contain `node_modules/` or any source `.tsx` files (`docker run --rm bamf-frontend:dev sh -c "find /app -name '*.tsx' | head -5"` returns nothing).
- `VITE_API_BASE_URL` is baked into the bundle; `docker run --rm bamf-frontend:dev sh -c "grep -r 'localhost:8000' /app/dist | head -1"` returns at least one match when the build arg is the default.
- The `HEALTHCHECK` directive is present.

## Test cases

- **TC-S001-NFR-002-01** (build): `docker build -t bamf-frontend:dev -f frontend/Dockerfile .` exits `0` on a clean checkout.
- **TC-S001-NFR-002-02** (run + serve): `docker run -d -p 3000:3000 --name bf bamf-frontend:dev && sleep 5 && curl -fsS http://localhost:3000/` returns the Vite-built `index.html` (status 200, body contains `<div id="root">`).
- **TC-S001-NFR-002-03** (asset): `curl -fsS http://localhost:3000/assets/$(docker exec bf ls /app/dist/assets | head -1)` returns 200.
- **TC-S001-NFR-002-04** (image size): `docker images bamf-frontend:dev --format "{{.Size}}"` reports under 200 MiB.
- **TC-S001-NFR-002-05** (no source in runtime): `docker run --rm bamf-frontend:dev sh -c "find /app -name '*.tsx' -o -name 'node_modules'"` returns nothing.
- **TC-S001-NFR-002-06** (registry override consumed): `docker build --build-arg NPM_REGISTRY=https://invalid.example.com/npm -f frontend/Dockerfile .` fails during `npm ci`.
- **TC-S001-NFR-002-07** (env baked): with `--build-arg VITE_API_BASE_URL=https://backend.acme.test`, `docker run --rm bamf-frontend:dev grep -ro 'backend.acme.test' /app/dist | head -1` returns at least one match.
- **TC-S001-NFR-002-08** (non-root): `docker run --rm bamf-frontend:dev whoami` outputs `app`.

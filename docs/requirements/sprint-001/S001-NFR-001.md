# Containerize the FastAPI backend with Artifactory-friendly Dockerfile

## Description

The backend currently has no Dockerfile (verified — `Dockerfile*` is absent at the project root). For BDOP deployment in the closed network, the backend must run from an image based on `${DOCKER_REGISTRY}/python:3.12-slim` and install Python dependencies through `${PIP_INDEX_URL}` (Artifactory PyPI proxy in production, public PyPI in dev). Resource limits target 2 vCPU / 2 GiB RAM as per the operator's hosting specification.

Add a new `backend/Dockerfile` (multi-stage where helpful, but a single stage is acceptable for the backend) that:

1. Uses `ARG DOCKER_REGISTRY=docker.io` so the registry path can be injected from `docker-compose build`.
2. `FROM ${DOCKER_REGISTRY}/python:3.12-slim`.
3. Sets `WORKDIR /app`, creates a non-root user `app` (UID 10001), and runs subsequent steps as that user.
4. Accepts `ARG PIP_INDEX_URL=https://pypi.org/simple` and runs `pip install --no-cache-dir --index-url "${PIP_INDEX_URL}" -r backend/requirements.txt`.
5. Copies `backend/` into `/app/backend/` excluding `venv/` and `__pycache__/` (handled by `.dockerignore`).
6. Copies `ontology_shacl_material/` into `/app/ontology_shacl_material/`.
7. Copies `root_docs/` into `/app/root_docs/` (used by S001-F-007's seed reset).
8. Copies `backend/data/contexts/templates/` (templates are required for `create_case_from_template`); does **not** copy `backend/data/contexts/cases/`.
9. Sets `ENV DOCUMENTS_PATH=/var/app/documents`, `ENV PYTHONUNBUFFERED=1`, `ENV LOG_LEVEL=INFO`.
10. Declares `EXPOSE ${BACKEND_PORT:-8000}` and runs `uvicorn backend.main:app --host 0.0.0.0 --port 8000`.
11. Adds a `HEALTHCHECK` that hits `GET /health` every 30s with a 5s timeout.

A new `.dockerignore` at the project root excludes: `.git/`, `node_modules/`, `dist/`, `backend/venv/`, `**/__pycache__/`, `**/.pytest_cache/`, `**/.mypy_cache/`, `*.env`, `info-env`, `backend/data/contexts/cases/ACTE-*`, `public/documents/ACTE-*`, `temp/`, `bamf-dev/`, `docs/code-graph/backups/`, the entire `docs/` tree (build artifacts not needed in the runtime image).

The image is **not** intended to run a database — `docs/database/current.md` confirms the project uses no traditional database. All persistence is on the filesystem, which is handled via volumes in S001-NFR-003.

The Dockerfile uses no multi-platform tricks (single `linux/amd64` target) and does not bake any secrets. `GEMINI_API_KEY`, `LITELLM_TOKEN`, and other sensitive values are passed at runtime via `docker-compose.yml`'s `env_file` directive.

## Acceptance criteria

- A file `backend/Dockerfile` exists.
- A file `.dockerignore` exists at the project root and contains every excluded pattern listed above.
- `docker build -t bamf-backend:dev --build-arg DOCKER_REGISTRY=docker.io --build-arg PIP_INDEX_URL=https://pypi.org/simple -f backend/Dockerfile .` succeeds and produces an image whose final size is under 800 MiB.
- The built image, run with `docker run --rm -p 8000:8000 -e GEMINI_API_KEY=dummy bamf-backend:dev`, listens on port 8000 within 15 seconds.
- `curl http://localhost:8000/health` against the running container returns `200 {"status": "healthy"}`.
- `docker inspect bamf-backend:dev --format '{{.Config.User}}'` returns `app` (non-root).
- The image does not contain any `.env`, `info-env`, or `backend/venv/` artifacts (`docker run --rm bamf-backend:dev ls /app/backend/venv` exits non-zero).
- `backend/data/contexts/templates/` is present in the image; `backend/data/contexts/cases/` is empty (or absent).
- `root_docs/` is present in the image with the six seed files.
- `ontology_shacl_material/` is present in the image.
- The `HEALTHCHECK` directive is present in the image config (`docker inspect` shows it).

## Test cases

- **TC-S001-NFR-001-01** (build): `docker build -t bamf-backend:dev -f backend/Dockerfile .` exits `0` on a clean checkout.
- **TC-S001-NFR-001-02** (run + health): `docker run -d --name bb -p 8000:8000 -e GEMINI_API_KEY=dummy bamf-backend:dev && sleep 15 && curl -fsS http://localhost:8000/health` returns `200`.
- **TC-S001-NFR-001-03** (no secrets baked): `docker history bamf-backend:dev` shows no layer with `GEMINI_API_KEY=...` or `LITELLM_TOKEN=...` baked in.
- **TC-S001-NFR-001-04** (non-root): `docker run --rm bamf-backend:dev whoami` outputs `app`.
- **TC-S001-NFR-001-05** (registry override): `docker build --build-arg DOCKER_REGISTRY=invalid.registry.example.com -f backend/Dockerfile .` fails with a registry-resolution error, proving the ARG is consumed.
- **TC-S001-NFR-001-06** (pip index override): `docker build --build-arg PIP_INDEX_URL=https://invalid.example.com/pypi/simple -f backend/Dockerfile .` fails during the pip install step, proving the ARG is consumed.
- **TC-S001-NFR-001-07** (image size): `docker images bamf-backend:dev --format "{{.Size}}"` reports under 800 MiB.
- **TC-S001-NFR-001-08** (healthcheck): `docker inspect bamf-backend:dev --format '{{.Config.Healthcheck.Test}}'` includes a `curl` or wget call to `/health`.

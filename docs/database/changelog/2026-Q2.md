## 2026-04-29 — Genesis (full build)

This entry records a full re-baseline of database documentation. All stores below are treated as "added" relative to a clean slate. Subsequent runs will diff against `current.md` granularly.

### Engines detected

- **No traditional database engine.** Graph confirms 0 `table` entities, 0 ORM imports (sqlalchemy / django / sqlmodel / mongoengine / pymongo / motor / psycopg / asyncpg / mysql.connector), 0 vector/search/cache client imports (pinecone / weaviate / qdrant / chroma / opensearch / elasticsearch / redis), 0 ORM-base `extends` relations, 0 database-shaped env vars (`DATABASE_URL`, `POSTGRES_*`, `MYSQL_*`, `MONGO_URI`, `REDIS_URL`, `PINECONE_*`, `OPENSEARCH_*`, `ELASTICSEARCH_*`).
- **Persistence model:** filesystem JSON stores + document blob hierarchy, with Python NamedTuple / dataclass / Pydantic models acting as the authoritative schema layer.

### Filesystem JSON Stores — Added

- **`document_manifest.json`** — defined in `backend/services/document_registry.py`. 4-field root (`version`, `schemaVersion`, `lastUpdated`, `documents[]`). `DocumentRegistryEntry` has 8 fields, `DocumentRender` has 5 fields. Logical cross-references to `caseId` (case dirs), `folderId` (folder_config), and the document blob hierarchy. Read by 8 functions, modified by 7 functions.
- **`case.json`** (per case) — defined in `backend/services/context_manager.py::ContextManager.load_case_context`. Container schema doc-stringed; `regulations[]` sub-schema authoritatively typed by `backend/models/regulation.py::Regulation` (5 declared fields + the schema-ahead-of-class `extractedText`).
- **`folder_config.json`** (per case) — defined in `backend/api/folders.py`. Pydantic-typed via `FolderConfigResponse` and `FolderConfig`. Auto-creates with 5 default folders (`personal-data`, `evidence`, `emails`, `certificates`, `applications`). Save invalidates the in-memory `_tree_view_cache` for the case.
- **`custom_rules.json`** (per case) — defined in `backend/api/custom_context.py`. Pydantic-typed via `CustomContextRule` (`Literal['validation_rule','required_document']` discriminator). 6 rule fields.
- **Per-folder context files** at `cases/{caseId}/folders/{folderId}.json` — defined in `backend/services/context_manager.py::ContextManager.load_folder_context`. Schema doc-stringed only (no class binding) — flagged as inferred.

### Document Blob Storage — Added

- **`{DOCUMENTS_BASE_PATH}/{caseId}/{folderId}/`** — case-scoped, folder-scoped binary file hierarchy. Original / render distinction via filename suffix (`__anonymized`, `__translated_{lang}`). Configurable via `DOCUMENTS_PATH` env var. Gated by `ENABLE_UPLOAD` feature flag (early 403 `feature_disabled` from `POST /api/files/upload` when off — S001-F-006).

### Read-Only Stores — Added

- **Case templates** (`backend/data/contexts/templates/{caseType}/`) — 3 templates: `asylum_application`, `family_reunification`, `integration_course`. Required for case creation; MUST be in image.
- **Ontology / SHACL reference** (`ontology_shacl_material/`) — 5 reference files (Turtle / JSON / Markdown / Mermaid).
- **Seed documents** (`root_docs/`) — 6 seed docs for the S001-F-007 reset flow.

### Volume Mapping — Added

`docker-compose.yml` declares three persistence volumes: `documents-data` → `/var/app/documents`, `cases-data` → `/app/backend/data/contexts/cases`, `manifest-data` → `/app/backend/data`. `ollama-data` is profile-gated and caches LLM weights, not application data.

### No critical / breaking changes

This is a genesis snapshot — no prior schema to break against. Subsequent changes will use the **[breaking]** marker for column-type changes, nullability flips, FK changes, vector dimension or metric changes, and search mapping type changes when applicable.

### Notes

- The graph does not yet emit access relations (`queries_table` / `modifies_table`) for filesystem-backed stores. Read/modified-by sets in `current.md` are derived from `defines` relations + authoritative-source review.
- `case.json::regulations[].extractedText` is currently a schema-ahead-of-class field (read by `backend/services/gemini_service.py::_load_context` but absent from `Regulation`'s declared dataclass fields — `Regulation.from_dict` ignores it). Intentional per S001-F-003.

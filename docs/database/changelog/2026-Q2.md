# Database Changelog — 2026 Q2

## 2026-04-27

**Genesis entry** — first run of `BO-db-documenter`. Everything below is "added" relative to no prior documented state.

### Engines detected

- No traditional database engine present (no SQL / NoSQL / vector / search / cache).
- Persistence implemented as JSON files on the local filesystem, plus a document blob store under `public/documents/` (configurable via `DOCUMENTS_PATH`).

### Persistent JSON stores — added

- **`document_manifest.json`** — defined in `backend/services/document_registry.py` (constant `MANIFEST_PATH`, line 39). Single source of truth for uploaded documents and their renders. Auto-created on first run; reconciled with filesystem at startup.
- **`case.json`** (per case at `backend/data/contexts/cases/{caseId}/case.json`) — defined in `backend/services/context_manager.py::ContextManager.load_case_context` (line 271). Created from template at case instantiation.
- **`folder_config.json`** (per case at `backend/data/contexts/cases/{caseId}/folder_config.json`) — defined in `backend/api/folders.py` (helper `get_folder_config_path`, line 78). Auto-created with 5 default folders if absent.
- **`custom_rules.json`** (per case at `backend/data/contexts/cases/{caseId}/custom_rules.json`) — defined in `backend/api/custom_context.py` (constant `CUSTOM_CONTEXT_PATH`, line 30).
- **Folder context files** (per case at `backend/data/contexts/cases/{caseId}/folders/{folderId}.json`) — defined in `backend/services/context_manager.py::ContextManager.load_folder_context` (line 332).

### Read-only templates — added

- **Case templates** at `backend/data/contexts/templates/{caseType}/` — defined in `backend/services/context_manager.py::ContextManager.create_case_from_template` (line 392). Three templates present: `asylum_application`, `family_reunification`, `integration_course`. **[breaking]** must be COPYed into Docker image; absence breaks case creation.

### Document blob storage — added

- **`{DOCUMENTS_PATH}/{caseId}/{folderId}/...`** — defined in `backend/config.py::get_documents_path` (line 98). Default `public/documents`; recommended container path `/var/app/documents`. Holds originals plus per-document renders (anonymized / translated). **[breaking]** must be a persistent Docker volume; uploads are lost without it.

### Read-only reference material — added

- **`ontology_shacl_material/`** — 5 files (`integration_course_ontology.ttl`, `integration_course_shacl.ttl`, `ontology_as_machine_readable.json`, `prompt_example.md`, `relationships.mmd`). Read by `backend/services/shacl_generator.py` and `backend/scripts/migrate_forms_to_shacl.py`. Must be COPYed into the Docker image.

### Environment variables — added (graph evidence)

`DOCUMENTS_PATH`, `GEMINI_API_KEY`, `IDIRS_BASE_URL`, `LOG_LEVEL`, `INIT_TEST_DOCS`, `SKIP_INTEGRATION_TESTS`, `VITE_API_URL`, `VITE_API_BASE_URL`.

### Environment Variables — new (LiteLLM proxy subproject, gitignored)

- `LITELLM_OLLAMA_HOST` — URL of the Ollama instance the proxy forwards to. Default: `http://host.docker.internal:11434`.
- `LITELLM_OLLAMA_MODEL` — documents the active model. Default: `gemma3:12b`. Note: changing also requires editing `litellm/config.yaml` (LiteLLM does not env-substitute `litellm_params.model`).

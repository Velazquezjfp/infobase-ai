## Engines Detected

This project uses **no traditional database** (no SQL, no NoSQL, no vector DB, no search engine, no Redis). All persistent state lives on the local filesystem as JSON files inside `backend/data/` and `public/documents/`.

- **Persistent JSON stores** (read-write at runtime)
- **JSON / Markdown templates** (read-only at runtime, copied at case creation)
- **Document blob storage** (read-write at runtime, hierarchy of binary files: PDFs, images, emails, certificates)
- **Static frontend assets** (read-only, bundled into `dist/` at build time)
- **Ontology / SHACL reference material** (read-only)

The graph confirms: 0 `table` entities, 0 ORM imports (no SQLAlchemy / Django / SQLModel / pymongo / redis / pinecone / opensearchpy). Persistence is implemented by hand in `backend/services/document_registry.py`, `backend/services/context_manager.py`, `backend/api/folders.py`, and `backend/api/custom_context.py`.

### Volumes that MUST persist across container restarts

1. `backend/data/contexts/cases/` — every per-case context, custom rule, folder config.

3. `backend/data/document_manifest.json` — the document registry index (lives inside the data volume).

If these are not mounted as volumes, **all user uploads, custom rules, and folder configurations are lost on container restart** — the registry will rebuild from filesystem (`reconcile()` in `document_registry.py`) but custom rules and folder configs cannot be recovered.

## Document Blob Storage (Read-Write at Runtime)

### Documents Hierarchy

**Base path (configurable):** `DOCUMENTS_PATH` env var, default `public/documents` (dev) or `/var/app/documents` (recommended for containers).
**Defined in:** `backend/config.py::get_documents_path` (line 98), `backend/api/folders.py::DOCUMENTS_DIR` (line 34).

**Directory layout:**

```
{DOCUMENTS_BASE_PATH}/
└── {caseId}/                # one subdir per case (e.g. ACTE-2024-001)
    ├── personal-data/       # one subdir per folder (matches folder_config.json IDs)
    ├── evidence/
    ├── emails/
    ├── certificates/
    ├── applications/
    ├── uploads/             # auto-created destination for files moved from deleted folders
    └── {folder-id}/
        ├── original.pdf
        ├── original__anonymized.pdf   # render
        └── original__translated_en.pdf # render
```

**File classification logic:** `document_registry.is_render_file` (line 523) determines whether a file is an original or a render based on filename suffix patterns.

**Mode:** Read-write. Files are uploaded by users via the upload endpoints, modified by anonymization/translation tools, and deleted via folder management.

**MUST be a Docker volume** — without persistence, all user uploads are lost on container restart. The manifest reconciler will detect missing files but cannot recover them.

## Document Blob Storage

**Base path (configurable):** `DOCUMENTS_PATH` env var. Default `public/documents` (dev), `/var/app/documents` (containers — set by `backend/Dockerfile` via `ENV`).
**Path resolvers:** `backend/config.py::get_documents_path` (line 98); `backend/api/folders.py::DOCUMENTS_DIR` (line 34).

### Directory layout

```
{DOCUMENTS_BASE_PATH}/
└── {caseId}/                # one subdir per case (e.g. ACTE-2024-001)
    ├── personal-data/       # one subdir per folder (matches folder_config.json IDs)
    ├── evidence/
    ├── emails/
    ├── certificates/
    ├── applications/
    ├── uploads/             # auto-created destination for files moved from deleted folders
    └── {folder-id}/
        ├── original.pdf
        ├── original__anonymized.pdf      # render
        └── original__translated_en.pdf   # render
```

**File classification:** `backend/services/document_registry.py::is_render_file` (line 523) determines original vs render based on filename suffix.

**Mode:** Read-write. Files are uploaded by users via upload endpoints, modified by anonymization/translation, and deleted via folder management.

**Feature gate (S001-F-006):** `POST /api/files/upload` early-returns `403 {"error": "feature_disabled"}` when `ENABLE_UPLOAD=false`. The handler body is intact behind the guard, so re-enabling is a single env-var flip. `GET /api/files/health.features.upload` mirrors the flag.

**Persistence requirement:** MUST be a Docker volume. Without persistence, all user uploads are lost on container restart. The manifest reconciler will detect missing files but cannot recover them. In `docker-compose.yml` this maps to the `documents-data` named volume.

## Warnings

- **No code-graph access traceability for JSON stores.** The graph does not yet emit `queries_table` / `modifies_table` relations for filesystem-backed stores. The "Read by" / "Modified by" lists above are derived from `defines` relations on the owning module plus authoritative-source review, not from inbound access relations. Expect false negatives if a caller in another module reads a store via a path-based helper that bypasses the documented service.
- **Per-folder context files have no class binding.** Schema is captured from a docstring; type drift can land silently.
- **`extractedText` field on regulations is ahead of the canonical dataclass.** `case.json::regulations[].extractedText` is read by `backend/services/gemini_service.py::_load_context` (introduced in S001-F-003 / Wave 2) but is not declared on `backend/models/regulation.py::Regulation` — `Regulation.from_dict` ignores it. Intentional, per Wave 2 design (operator-prepared offline regulation body); flagged here so future schema-strict refactors do not silently drop it.
- **Staleness:** working tree dirty at run time (graph backups + regenerated graph). Graph mtime is newer than HEAD commit, so the graph reflects working-tree state — fine for documentation, but downstream readers should be aware.
